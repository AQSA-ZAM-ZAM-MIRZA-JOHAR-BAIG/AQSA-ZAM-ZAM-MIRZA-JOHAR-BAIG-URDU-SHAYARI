## ❌ Common Mistakes to Avoid

1. **Missing Schema Update**: Forgetting to update `app/layout.tsx` with full Person schema
   - **Impact**: Lose entity authority signals
   - **Fix**: Copy schema from guide immediately after creating it

2. **Not Testing Before Deploy**: Deploying without `npm run build`
   - **Impact**: Build failures on Vercel
   - **Fix**: Always test locally first (5 min)

3. **Forgetting GSC Sitemap**: Creating sitemap but not submitting to GSC
   - **Impact**: Pages delayed in indexing
   - **Fix**: Submit within 1 hour of deployment

4. **Writing Blog Posts Too Early**: Starting content expansion before Phase 0 links are in place
   - **Impact**: New blog content dilutes PageRank flow
   - **Fix**: Wait for Phase 1 after Phase 0 is deployed and indexed

5. **Changing Content Too Frequently**: Updating pages multiple times per week
   - **Impact**: Canonical confusion, ranking volatility
   - **Fix**: Deploy Phase 0, wait 1 week for indexing, then start Phase 1 updates

## 📞 Support: If Something Goes Wrong

**Issue**: Build fails after deploying
- **Solution**: Run `npm run build` locally to debug, check error messages

**Issue**: Pages not showing in GSC after 3 days
- **Solution**: Click "Request Indexing" in GSC for homepage only (others follow via sitemap)

**Issue**: Schema validation errors
- **Solution**: Use https://validator.schema.org/ to debug JSON-LD

**Issue**: Robots.txt blocking Google
- **Solution**: Verify with https://tools.seobook.com/robots-txt-generator
